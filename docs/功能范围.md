# 功能范围

## 1. 产品一句话

给情侣用的私密旅行地图与旅行计划网页：用一张中国地图记录去过哪里，用图文日记保存每段旅行，用日历规划下一次出发。

## 2. 目标用户

情侣。

## 3. 核心场景

1. 两个人打开共同主页，看到一起去过的中国地图。
2. 点击去过的地区，进入那段旅行的照片和日记。
3. 两个人都可以发布图片和文字组合的旅行记录。
4. 两个人可以一起编辑未来旅行计划。
5. 通过邀请链接或邀请码，让另一半加入共同空间。

## 4. 第一版必须做

### 4.1 地图首页

1. 首页是一整张中国地图。
2. 地图可以缩放。
3. 去过的地区有特殊标注。
4. 去过的地区显示一张旅行代表照片。
5. 点击地区进入该地区的旅行记录页面。
6. 第一版使用 Apache ECharts 实现地图。
7. 初始展示全国省份地图，点击省份后下钻进入该省城市地图。
8. 城市地图中展示去过城市的记录标记和代表照片。
9. 缩放只用于放大查看地图细节，不作为自动切换省份/城市层级的触发条件。
10. 第一版不展示道路和街道，不提供定位及路线导航。

### 4.2 旅行记录

1. 可以创建旅行。
2. 可以写旅行日记。
3. 可以上传照片。
4. 图片和文字可以组合发送。
5. 记录流呈现方式类似朋友圈。
6. 两个人都可以发布和编辑旅行记录。
7. 第一版采用一个城市一个记录流，不区分第几次去同一城市。
8. 旅行记录页面上方需要有日期范围选择，例如 `2026年6月1日` 到 `2026年6月3日`。
9. 一条图文旅行记录最多可以上传 6 张图片。

### 4.3 旅行计划

1. 计划页面使用日历形态。
2. 点击某一天可以编辑当天计划。
3. 计划内容包括时间安排、每天去哪。
4. 编辑页面预留语音输入入口。

### 4.4 双人互通

1. 有情侣空间一样的共同主页。
2. 一个人创建空间，另一个人通过邀请加入。
3. 两个人可以共同上传照片、写日记、编辑旅行和计划。
4. 情侣空间是共享内容容器，地图记录、照片、日记、计划都归属于同一个 `space_id`。
5. 任意一方发布或修改内容后，另一方进入同一情侣空间可以看到。

### 4.6 确认 / 取消交互

1. 新增或编辑内容时，不做边输入边保存。
2. 点击 `确定` 后才保存修改。
3. 点击 `取消` 后放弃本次修改或发送。
4. 图文日记发布、旅行计划编辑都必须有明确确认动作。

### 4.7 使用方式

1. 手机端必须可用。
2. 先不登录也能体验核心流程。
3. 内容默认完全私密。

## 5. 第一版暂不做

1. 地图导航。
2. 机票酒店接口。
3. 公开社区。
4. 评论点赞。
5. AI 自动规划。
6. App 上架。

## 6. 第一版完成标准

一周内完成可点击、可演示的网页原型：

1. 可以打开地图首页。
2. 可以看到已去过地区标注。
3. 可以点击地区进入记录流。
4. 可以新增一条图文记录。
5. 可以打开日历计划页。
6. 可以点击日期编辑计划。
7. 可以看到邀请另一半加入的入口。
8. 手机尺寸下布局正常。
