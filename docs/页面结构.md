# 页面结构

## 1. 页面清单

第一版建议先做 6 个页面：

| 页面 | 作用 |
| --- | --- |
| 地图首页 | 展示中国地图、去过区域、代表照片 |
| 地区旅行记录页 | 展示某个地区的照片和日记记录流 |
| 新增图文记录页 | 发布图片和文字组合记录 |
| 日历计划页 | 查看旅行计划日历 |
| 日期计划编辑页 | 编辑某一天的时间安排和要去哪里 |
| 我们 / 绑定页面 | 创建情侣空间、输入邀请码、查看绑定状态 |

## 2. 地图首页

核心内容：

1. 可缩放地图，体验参考百度地图。
2. 初始显示全国省份。
3. 点击省份后下钻进入该省城市地图。
4. 缩放只用于放大查看地图细节，不自动切换省份/城市层级。
5. 已去过地区高亮。
6. 地区代表照片。
7. 地区名称。
8. 底部主导航：地图、计划、记录、我们。

交互：

1. 缩放地图。
2. 点击省份进入省内城市地图。
3. 点击城市进入地区旅行记录页。
4. 点击返回全国回到中国省份地图。
5. 点击加号创建新旅行。
6. 如果用户还没有绑定情侣空间，首次进入时显示绑定引导。

## 3. 地区旅行记录页

核心内容：

1. 地区名，例如青岛。
2. 旅行日期范围选择，例如 `2026年6月1日` 到 `2026年6月3日`。
3. 代表照片。
4. 图文记录流。
5. 第一版一个城市只有一个记录流，不区分第几次旅行。
6. 第一版支持从已上传图片中手动设为城市代表照片。

记录卡片包含：

1. 发布人。
2. 发布时间。
3. 图片。
4. 文字日记。
5. 关联的旅行日期范围。

交互：

1. 从地图点击地区后进入该地区记录页面。
2. 当前地区记录按 `space_id + region` 查询。
3. 两个人在同一地区发布的图片和日记都展示在同一个记录流中。
4. 页面只保留一个主要操作按钮：`发布记录`。
5. 修改日期范围、发布记录都需要点击 `确定` 或 `发布` 后才保存。
6. 点击 `取消` 放弃本次编辑。
7. 点击某张已上传图片的 `设为封面` 后，将该图片保存为当前城市记录流的代表照片。

## 4. 新增图文记录页

核心内容：

1. 选择或上传图片。
2. 输入日记文字。
3. 选择或确认关联城市。
4. 选择旅行日期范围。
5. 发布按钮。

交互：

1. 点击 `发布` 或 `确定` 后才保存。
2. 点击 `取消` 后放弃本次输入。
3. 输入过程中不自动写入数据库。
4. 图片和日记在同一个界面填写，并作为同一条记录一起发布。
5. 第一版一条记录最多 6 张图片。
6. 发布成功后，用户可以在记录图片上选择 `设为封面`。

后续增强：

1. 语音输入转文字。
2. 自动提取照片时间和地点。

## 5. 日历计划页

核心内容：

1. 月历。
2. 有计划的日期标记。
3. 当天计划摘要。

交互：

1. 点击某一天进入日期计划编辑页。
2. 新增旅行计划。

## 6. 日期计划编辑页

核心内容：

1. 日期。
2. 时间安排。
3. 每天去哪。
4. 备注。
5. 语音输入入口。

第一版语音输入可以先做成按钮占位，后续再接浏览器语音识别能力。

交互：

1. 点击 `确定` 后保存计划。
2. 点击 `取消` 后放弃本次修改。
3. 计划保存后，同一个情侣空间的另一方可以看到。

## 7. 我们 / 绑定页面

未绑定时显示：

1. 创建我们的旅行空间。
2. 输入邀请码加入。

已创建空间但对方未加入时显示：

1. 当前邀请码。
2. 有效期。
3. 复制邀请码。
4. 重新生成邀请码。

已绑定时显示：

1. 情侣空间名称。
2. 成员信息。
3. 一起去过的地区数量。
4. 进入地图或记录的入口。
