# 部署配置清单

## 1. 文档用途

这份文档用于整理 `project-001-love-travel-map` 上线时必须准备的环境变量和外部服务配置。

原则：

1. 真实密码、AccessKey、API Key 不写进代码和文档。
2. 线上配置优先放在部署平台的环境变量、服务器系统环境变量或 `.env` 文件中。
3. `.env` 文件只放服务器本机，不提交到 Git。
4. 前端只允许出现 `NEXT_PUBLIC_` 开头且可以公开的配置。
5. Java 后端统一作为业务入口，Python AI 服务不直接暴露给前端用户。

> 注意：之前已经在聊天中出现过的云厂商 AccessKey，正式上线前建议在阿里云控制台轮换一次，避免后期开源或截图时留下安全隐患。

## 2. MySQL 配置

| 配置项 | 环境变量 | 本地示例 | 线上填写 |
| --- | --- | --- | --- |
| MySQL 地址和数据库名 | `MYSQL_URL` | `jdbc:mysql://localhost:3306/love_travel?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai` | `jdbc:mysql://服务器内网地址:3306/love_travel?...` |
| MySQL 账号 | `MYSQL_USERNAME` | `root` | 建议使用项目专用账号 |
| MySQL 密码 | `MYSQL_PASSWORD` | 本地密码 | 线上强密码 |
| 数据库名 | `love_travel` | `love_travel` | `love_travel` |

建议：

1. 线上不要用 MySQL `root` 账号跑业务。
2. 创建项目专用账号，只授予 `love_travel` 数据库需要的权限。
3. 上线前确认所有 SQL 迁移脚本已经执行完成。

## 3. Redis 配置

| 配置项 | 环境变量 | 本地示例 | 线上填写 |
| --- | --- | --- | --- |
| Redis 地址 | `REDIS_HOST` | `localhost` | 服务器内网地址或 `127.0.0.1` |
| Redis 端口 | `REDIS_PORT` | `6379` | `6379` 或实际端口 |
| Redis 密码 | `REDIS_PASSWORD` | 留空或本地密码 | 线上强密码 |

当前项目 Redis 用途：

1. 登录 Session。
2. AI 规划额度统计。
3. AI 规划防重复点击和限流。
4. 后续可扩展邀请码冷却、热点缓存、分布式锁。

部署要求：

1. Redis 不直接暴露公网。
2. 线上必须设置密码。
3. 如果 Redis 和 Java 后端在同一台服务器，优先只监听本机或内网。

## 4. OSS 配置

| 配置项 | 环境变量 | 示例 | 说明 |
| --- | --- | --- | --- |
| COS Region | `TENCENT_COS_REGION` | `ap-guangzhou` | 按 bucket 所在地域填写 |
| COS Bucket | `TENCENT_COS_BUCKET` | `love-travel-images-jyh-1447356754` | 当前项目图片 bucket 完整名称 |
| SecretId | `TENCENT_COS_SECRET_ID` | 不写入文档 | 只放后端环境变量 |
| SecretKey | `TENCENT_COS_SECRET_KEY` | 不写入文档 | 只放后端环境变量 |
| COS 公共访问前缀 | `TENCENT_COS_PUBLIC_BASE_URL` | 可留空 | 私有 bucket 留空，后端返回临时签名 URL |
| 签名链接过期分钟数 | `TENCENT_COS_SIGNED_URL_EXPIRE_MINUTES` | `10` | 私有 bucket 推荐 10 分钟 |

安全要求：

1. AccessKey 只能配置在 Java 后端。
2. 前端不能保存 AccessKey。
3. 删除图片时后端同步删除 OSS 对象，避免存储费用堆积。
4. 正式上线建议使用 RAM 子账号，只授予当前 bucket 的必要权限。

## 5. DashScope 配置

| 配置项 | 环境变量 | 本地示例 | 说明 |
| --- | --- | --- | --- |
| 百炼 API Key | `ALIYUN_DASHSCOPE_API_KEY` | 不写入文档 | 只放 Python AI 服务环境变量 |
| 通义模型名称 | `QWEN_MODEL_NAME` | `qwen-plus` | 当前 AI 旅行规划使用 |

当前调用链路：

```text
前端计划页
  -> Java 后端 /api/ai/plan-days/{dayId}/generate-stream
  -> Python AI 服务 /internal/ai/plan-day/generate-stream
  -> DashScope qwen-plus
```

要求：

1. Python AI 服务不直接暴露给公网用户。
2. Java 后端负责登录、空间权限、额度、防重复点击。
3. Python 只负责模型调用和流式生成。

## 6. Java 后端配置

| 配置项 | 环境变量 | 本地示例 | 线上示例 |
| --- | --- | --- | --- |
| Java 后端端口 | `SERVER_PORT` | `8080` | `8080` |
| 登录 Session 密钥 | `LOVE_TRAVEL_SESSION_SECRET` | 本地可用默认值 | 必须改成强随机字符串 |
| Python AI 服务地址 | `AI_SERVICE_BASE_URL` | `http://localhost:8000` | `http://127.0.0.1:8000` |
| 允许访问后端的前端域名 | `FRONTEND_ALLOWED_ORIGINS` | `http://127.0.0.1:3000,http://localhost:3000` | `https://你的前端域名` |

说明：

1. `SERVER_PORT` 已支持通过环境变量修改。
2. `FRONTEND_ALLOWED_ORIGINS` 多个域名用英文逗号分隔。
3. 因为登录使用 Cookie，线上前端域名必须加入 CORS 白名单。

## 7. Python AI 服务配置

| 配置项 | 环境变量 | 本地示例 | 线上示例 |
| --- | --- | --- | --- |
| Python AI 服务端口 | 启动命令指定 | `8000` | `8000` |
| 百炼 API Key | `ALIYUN_DASHSCOPE_API_KEY` | 不写入文档 | 平台或服务器环境变量 |
| 通义模型名称 | `QWEN_MODEL_NAME` | `qwen-plus` | `qwen-plus` |

本地启动示例：

```powershell
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

线上建议：

```text
监听 127.0.0.1:8000
只允许 Java 后端本机调用
不开放 8000 到公网
```

## 8. 前端配置

| 配置项 | 环境变量 | 本地示例 | 线上示例 |
| --- | --- | --- | --- |
| 前端访问的 Java 后端 API 地址 | `NEXT_PUBLIC_API_BASE_URL` | `http://127.0.0.1:8080` | `https://api.example.com` |

说明：

1. `NEXT_PUBLIC_API_BASE_URL` 会被打包进前端代码，因此只能放公开地址，不能放任何密钥。
2. 如果前后端同域名并由 Nginx 转发 `/api`，可以考虑让前端请求相对路径，减少跨域和 Cookie 问题。
3. 如果前端部署在 Netlify / Cloudflare Pages，后端必须配置 `FRONTEND_ALLOWED_ORIGINS`。

## 10. 部署方式建议

### 10.1 免费试水

先只部署前端到 Netlify 或 Cloudflare Pages。

适合场景：

1. 先体验前端上线流程。
2. 看看域名、构建、环境变量、页面访问是否正常。
3. 不追求完整登录、图片上传、AI 规划真实可用。

限制：

1. Java 后端、Python AI 服务、MySQL、Redis 仍在本地或暂未部署。
2. 真正业务链路不会完整可用。
3. 如果后端仍在本地，公网用户无法访问本地后端。

### 10.2 真正可用

使用阿里云轻量应用服务器部署全套。

推荐部署方式：

```text
Nginx
  -> Next.js 前端
  -> Spring Boot Java 后端
  -> FastAPI Python AI 服务
  -> MySQL
  -> Redis
  -> 腾讯云 COS
  -> DashScope
```

适合场景：

1. 想让别人真正注册、登录、上传照片、使用 AI 规划。
2. 想练企业项目部署链路。
3. 想把项目作为面试项目讲完整。

第一版推荐结论：

```text
先做免费前端试水，熟悉部署平台和构建变量。
再用阿里云轻量服务器部署全套，形成真正可演示的完整项目。
```
