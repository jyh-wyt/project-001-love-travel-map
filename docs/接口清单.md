# 接口清单

更新时间：2026-06-17

## 1. 接口开发规则

本项目后续新增任何业务接口前，必须先和用户确认：

1. 前端页面入口和按钮行为。
2. 用户点击 `确定`、`取消`、`发布`、`删除` 时分别发生什么。
3. 请求参数、返回字段、错误提示。
4. 数据保存位置和是否逻辑删除。
5. 情侣空间共享规则和权限边界。

默认开发顺序：

```text
讨论功能 -> 确认前端交互 -> 确认接口契约 -> 编写后端接口 -> 联调验证
```

## 2. 当前已实现接口

### 2.1 账号登录

| 页面 | 前端动作 | 方法 | 路径 | 状态 |
| --- | --- | --- | --- | --- |
| 登录页 | 注册账号 | `POST` | `/api/auth/register` | 已实现 |
| 登录页 | 登录账号 | `POST` | `/api/auth/login` | 已实现 |

注册请求体：

```json
{
  "account": "qingdao2026",
  "password": "123456",
  "confirmPassword": "123456",
  "nickname": "焦焦"
}
```

登录请求体：

```json
{
  "account": "qingdao2026",
  "password": "123456"
}
```

返回：

```json
{
  "userId": 1,
  "account": "qingdao2026",
  "nickname": "焦焦"
}
```

规则：

1. 前端不再自动创建匿名用户。
2. 未登录访问地图、计划、空间、城市记录时，会跳转到 `/login`。
3. 登录成功后，前端把当前用户保存在 localStorage。
4. 账号不能重复，只能使用英文字母、数字和下划线，长度 5 到 20 位。
5. 用户名不能重复，长度 2 到 20 个字符；当前保存在 `app_user.nickname` 字段。
6. 密码至少 8 位，后端使用 BCrypt 保存密码哈希，不保存明文密码。
7. 注册时必须再次输入确认密码，两次密码一致才允许注册。
8. 账号数据保存在 MySQL 的 `app_user` 和 `user_auth_identity` 表中，只要数据库不删除，账号会一直有效。
9. 第一版使用账号密码登录，后续可扩展手机号、邮箱或微信登录。

| --- | --- | --- | --- | --- |
| 情侣空间页 | 查询当前空间 | `GET` | `/api/spaces/current?userId={userId}` | 已实现 |
| 情侣空间页 | 创建空间 | `POST` | `/api/spaces` | 已实现 |
| 情侣空间页 | 生成邀请码 | `POST` | `/api/spaces/{spaceId}/invite-code` | 已实现 |

2. 未绑定情侣时，用户也可以单人使用地图和计划。

5. 如果两边都有同一个城市的记录，会合并到同一个城市记录流。

### 2.3 计划页

| 页面 | 前端动作 | 方法 | 路径 | 状态 |
| --- | --- | --- | --- | --- |
| 计划页 | 获取全部计划天数 | `GET` | `/api/plans/days?userId={userId}` | 已实现 |
| 计划页 | 新增一天 | `POST` | `/api/plans/days` | 已实现 |
| 计划页 | 保存当天修改 | `PUT` | `/api/plans/days/{dayId}` | 已实现 |
| 计划页 | 删除一天 | `DELETE` | `/api/plans/days/{dayId}?userId={userId}` | 已实现 |

请求体：

```json
{
  "userId": 1,
  "date": "2026-06-19",
  "title": "到达青岛",
  "detail": "下午到达，晚上去海边。"
}
```

规则：

1. 点击 `新增一天` 会创建一条空计划。
2. 点击 `确定保存当天` 才保存编辑内容。
3. 点击 `取消` 不调用保存接口。
4. 删除前必须弹出确认框。

### 2.4 地图和城市记录

| 页面 | 前端动作 | 方法 | 路径 | 状态 |
| --- | --- | --- | --- | --- |
| 首页中国地图 | 获取已去过省份/城市 | `GET` | `/api/map/visited-regions?userId={userId}` | 已实现 |
| 省内城市地图 | 获取省内已去过城市 | `GET` | `/api/map/visited-regions?userId={userId}&provinceCode={provinceCode}` | 已实现 |
| 城市记录页 | 进入城市记录流 | `GET` | `/api/regions/{cityCode}/travel?userId={userId}&provinceCode={provinceCode}&provinceName={provinceName}&cityName={cityName}` | 已实现 |
| 城市记录页 | 保存旅行日期 | `PUT` | `/api/trips/{tripId}/date-range` | 已实现 |
| 城市记录页 | 设置封面图 | `PUT` | `/api/trips/{tripId}/cover-image` | 已实现 |

规则：

1. 地图标记只统计有未删除日记记录的城市。
2. 如果某城市所有日记都被删除，地图上该城市不再显示为已去过。
3. 城市记录日期只在上方保存一次，发布日记时不再重复选择日期。
4. 封面默认使用第一张未删除图片，也可以手动设置。

### 2.5 图文日记

| 页面 | 前端动作 | 方法 | 路径 | 状态 |
| --- | --- | --- | --- | --- |
| 城市记录页 | 发布图文日记 | `POST` | `/api/trips/{tripId}/posts` | 已实现 |
| 城市记录页 | 编辑日记文字并追加图片 | `PUT` | `/api/posts/{postId}` | 已实现 |
| 城市记录页 | 删除整条记录 | `DELETE` | `/api/posts/{postId}?userId={userId}` | 已实现 |
| 城市记录页 | 删除单张图片 | `DELETE` | `/api/images/{imageId}?userId={userId}` | 已实现 |

发布请求体：

```json
{
  "userId": 1,
  "content": "今天去了海边。",
  "images": [
    {
      "imageUrl": "signed-url-for-preview",
      "ossObjectKey": "travel/1/2026/06/xxx.jpg",
      "sortOrder": 0
    }
  ]
}
```

编辑请求体：

```json
{
  "userId": 1,
  "content": "修改后的日记内容"
}
```

规则：

1. 一条记录最多 6 张图片。
2. 图片和日记可以一起发布。
3. 点击 `发布` 后才保存。
4. 点击 `取消` 不保存。
5. 删除记录和删除图片都必须二次确认。
6. 删除图片或记录时，后端会同步删除 OSS 对象。

### 2.6 OSS 图片

| 页面 | 前端动作 | 方法 | 路径 | 状态 |
| --- | --- | --- | --- | --- |
| 城市记录页 | 检查 OSS 配置 | `GET` | `/api/oss/status` | 已实现 |
| 城市记录页 | 上传图片 | `POST` | `/api/oss/images` | 已实现 |

上传请求：

```text
multipart/form-data
userId: 1
file: image file
```

规则：

1. 单张图片最大 10MB。
2. 支持 JPG、PNG、WebP、GIF。
3. Bucket 建议保持私有。
4. 数据库存 `ossObjectKey`，前端展示使用后端返回的临时签名 URL。
5. AccessKey 只能放在后端环境变量，不能写进代码。
