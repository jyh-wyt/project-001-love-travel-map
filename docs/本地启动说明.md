# 本地启动说明

更新时间：2026-06-17

## 1. 启动顺序

在项目根目录执行：

```cmd
scripts\start-all.cmd
```

它会依次启动：

1. Redis
2. Python AI 服务
3. Java 后端
4. Next.js 前端

## 2. 访问地址

启动后访问：

| 服务 | 地址 |
| --- | --- |
| 前端 | `http://127.0.0.1:3000` |
| Java 后端 | `http://127.0.0.1:8080` |
| AI 服务 | `http://127.0.0.1:8000/internal/health` |

## 3. 一次性完整验证流程

按这个顺序测：

1. 打开前端首页。
2. 如果跳到登录页，先注册或登录账号。
3. 打开“情侣空间”页。
4. 创建个人空间或直接获取邀请码。
5. 生成邀请码。
6. 退出当前账号，用另一个账号登录后输入邀请码加入空间。
7. 打开“计划”页。
8. 新增一天，填写标题和安排，点保存。
9. 删除一天，确认弹窗后删除。
10. 打开“中国地图”首页。
11. 点击已去过省份。
12. 打开城市页。
13. 先保存旅行日期。
14. 上传图片。
15. 填写日记并发布。
16. 编辑已发布日记。
17. 删除图片或整条记录，确认弹窗后删除。

## 4. 你应该看到的结果

1. 城市记录发布后，地图上对应城市会被标记为已去过。
2. 如果把该城市所有记录都删除，城市标记会恢复正常。
3. 计划页的新增、修改、删除都只在确认后生效。
4. 图片上传成功后会显示在待发布区。
5. 删除图片时会同步删 OSS 对象。

## 5. 单独启动

### 5.1 Redis

```cmd
scripts\start-redis.cmd
```

### 5.2 AI 服务

```cmd
scripts\start-ai-service.cmd
```

### 5.3 Java 后端

```cmd
scripts\start-server.cmd
```

### 5.4 前端

```cmd
scripts\start-front.cmd
```

前端启动命令固定为：

```cmd
next dev --hostname 127.0.0.1 --port 3000 --webpack
```

原因：当前环境下这样更稳。

## 6. 常见问题

### 6.1 前端 3000 端口被占用

说明：先关掉已经启动的 Next.js，再重新执行启动脚本。

### 6.2 后端启动后页面没内容

先确认：

1. Java 后端 8080 端口可访问。
2. MySQL 和 Redis 已启动。
3. OSS 环境变量已配置。

### 6.3 图片上传失败

检查：

1. OSS 环境变量是否配置完整。
2. Bucket 是否存在。
3. AccessKey 是否有效。
4. 图片是否超过 10MB。

### 6.4 登录失败

检查：

1. 前端是否能访问 8080。
2. 后端是否已启动。
3. 账号或密码是否填写正确。

## 7. 说明

2. AI 服务的健康检查是 `/internal/health`。
3. 当前第一版核心流程已经可以联调。
