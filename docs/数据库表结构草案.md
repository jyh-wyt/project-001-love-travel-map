# 数据库表结构草案

## 1. 设计目标

第一版数据库只服务 MVP，不做过度设计。

需要支持：

1. 可升级账号体系。
2. 情侣空间。
3. 邀请码加入。
4. 旅行记录。
5. 图文日记。
6. 图片元数据。
7. 日历计划。
8. AI 日记润色记录。

## 2. 表清单

第一版建议 10 张核心表：

| 表名 | 作用 |
| --- | --- |
| `app_user` | 用户主表 |
| `user_auth_identity` | 用户登录身份表 |
| `couple_space` | 情侣空间 |
| `couple_space_member` | 情侣空间成员 |
| `invite_code` | 邀请码 |
| `trip` | 一次旅行 |
| `trip_post` | 旅行图文日记 |
| `trip_image` | 旅行图片 |
| `travel_plan_day` | 日历计划 |
| `ai_polish_log` | AI 日记润色记录 |

## 3. 账号体系设计原则

第一版使用：

```text
匿名体验 + 邀请码
```

但底层不使用只能支持匿名的表结构，而是从一开始设计成可升级账号体系。

这样后续增加邮箱登录、手机号登录、微信登录时，不需要推翻情侣空间、旅行、图片、日历计划和 AI 记录等业务表。

## 4. `app_user` 用户主表

用于保存用户基础信息。第一版匿名用户也写入这张表。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | BIGINT | 主键 |
| `nickname` | VARCHAR(50) | 昵称，可选 |
| `avatar_url` | VARCHAR(500) | 头像，可选 |
| `user_type` | VARCHAR(20) | 用户类型：anonymous / registered |
| `status` | VARCHAR(20) | 状态：active / disabled |
| `created_at` | DATETIME | 创建时间 |
| `updated_at` | DATETIME | 更新时间 |
| `deleted` | TINYINT | 逻辑删除 |

## 5. `user_auth_identity` 用户登录身份表

用于保存用户的登录身份。一个用户后续可以绑定多个身份。

第一版匿名用户：

```text
identity_type = anonymous
identity_value = 浏览器生成的 anonymous_id
```

后续可扩展：

```text
identity_type = email
identity_type = phone
identity_type = wechat
```

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | BIGINT | 主键 |
| `user_id` | BIGINT | 用户 ID |
| `identity_type` | VARCHAR(30) | 身份类型：anonymous / email / phone / wechat |
| `identity_value` | VARCHAR(200) | 身份值，如 anonymous_id、邮箱、手机号、openid |
| `credential_hash` | VARCHAR(255) | 凭证哈希，可选，例如密码哈希 |
| `verified` | TINYINT | 是否已验证 |
| `created_at` | DATETIME | 创建时间 |
| `updated_at` | DATETIME | 更新时间 |
| `deleted` | TINYINT | 逻辑删除 |

| `updated_at` | DATETIME | 更新时间 |

## 10. `trip_post` 旅行图文日记表

用于保存像朋友圈一样的图文记录。

第一版限制：

```text
一条图文记录最多 6 张图片。
```

发布图文记录时，旅行日期范围可选。城市代表照片优先使用用户手动设置的封面；如果未设置，可以默认展示第一条记录的第一张图片。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | BIGINT | 主键 |
| `space_id` | BIGINT | 情侣空间 ID |
| `trip_id` | BIGINT | 旅行 ID |
| `author_user_id` | BIGINT | 作者用户 ID |
| `content` | TEXT | 日记正文 |
| `polished_content` | TEXT | AI 润色后的内容，可选 |
| `post_date` | DATE | 记录日期 |
| `location_name` | VARCHAR(100) | 地点名称，可选 |
| `created_at` | DATETIME | 创建时间 |
| `updated_at` | DATETIME | 更新时间 |
| `deleted` | TINYINT | 逻辑删除 |

## 14. 后续登录升级方式

第一版匿名用户升级成正式账号时：

1. 保留原 `app_user` 记录。
2. 在 `user_auth_identity` 中新增邮箱、手机号或微信身份。
3. 将 `app_user.user_type` 从 `anonymous` 改为 `registered`。
4. 原情侣空间、旅行记录、图片、计划、AI 日志全部继续关联同一个 `user_id`，不需要迁移。

## 15. 待确认问题

以下问题已确认：

1. 邀请码有效期：24 小时。
2. 第一版情侣空间严格限制 2 人。
3. 匿名用户身份用浏览器 localStorage 保存。
4. 第一版图片不做缩略图。
5. AI 润色日志保存完整输入和输出。
