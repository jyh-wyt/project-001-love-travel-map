# 前端免费部署步骤

更新时间：2026-06-25

## 1. 当前建议

当前第一步只做前端免费试水部署，不急着部署 Java 后端、Python AI 服务、MySQL 和 Redis。

推荐先用：

```text
Netlify
```

原因：

1. 当前你没有 Vercel 需要的手机号条件。
2. Netlify 支持 Next.js 项目部署。
3. 可以连接 GitHub 仓库，后续推送代码后自动重新部署。
4. 免费层足够用来练习前端上线流程。

## 2. 这一步能得到什么

部署成功后，你会得到一个公网前端地址，例如：

```text
https://love-travel-map.netlify.app
```

别人可以打开这个地址看到前端页面。

但要注意：

1. Java 后端还没部署时，登录、上传图片、AI 规划等真实接口不能完整使用。
2. 这一步是“前端上线试水”，不是完整项目上线。
3. 完整可用还需要部署 Java 后端、Python AI 服务、MySQL、Redis，并配置 OSS 和 DashScope。

## 3. 当前项目已准备好的内容

项目根目录已经新增：

```text
netlify.toml
```

内容作用：

```text
base = apps/front
command = npm run build
publish = .next
NODE_VERSION = 22
```

并且要让 Netlify 明确启用 Next.js 运行时插件：

```text
[[plugins]]
  package = "@netlify/plugin-nextjs"
```

本地已经验证：

```text
npm.cmd run build
```

构建通过。

## 4. 前置准备

你需要准备：

1. GitHub 账号。
2. Netlify 账号，可以用 GitHub 登录。
3. 把 `project-001-love-travel-map` 推送到 GitHub 仓库。

建议仓库结构保持：

```text
project-001-love-travel-map/
  apps/
    front/
    server/
    ai-service/
  docs/
  data/
  netlify.toml
  README.md
```

## 5. 推送 GitHub 前安全检查

推送前必须确认：

```text
[ ] 没有提交 OSS AccessKey
[ ] 没有提交 DashScope API Key
[ ] 没有提交 MySQL 密码
[ ] 没有提交 Redis 密码
[ ] 没有提交 `.env`
[ ] 没有提交本地日志里的敏感信息
[ ] 文档中没有真实 secret
```

注意：

```text
真实密钥只放部署平台环境变量或服务器环境变量。
不要写进代码、README、docs、截图。
```

## 6. Netlify 部署步骤

### 第一步：打开 Netlify

访问：

```text
https://app.netlify.com/
```

使用 GitHub 登录。

### 第二步：导入项目

进入 Netlify 后选择：

```text
Add new site
  -> Import an existing project
  -> Deploy with GitHub
```

授权后选择你的 GitHub 仓库：

```text
project-001-love-travel-map
```

### 第三步：确认构建配置

因为项目根目录已经有 `netlify.toml`，Netlify 通常会自动读取。

如果页面要求手动确认，填写：

```text
Base directory: apps/front
Build command: npm run build
Publish directory: .next
```

Node 版本：

```text
22
```

注意：

```text
如果部署成功但打开预览地址或永久链接显示 Netlify Page not found，
优先检查本次 Deploy log 里是否出现 Next.js Runtime / OpenNext / @netlify/plugin-nextjs。
如果没有出现，说明 Netlify 可能把 .next 当成普通静态目录发布了，需要确认仓库根目录的 netlify.toml 已经提交并推送。
```

### 第四步：配置前端环境变量

当前只做前端免费试水时，可以先不配置后端地址。

如果 Netlify 要你填，后端没部署前可以临时不填：

```text
NEXT_PUBLIC_API_BASE_URL
```

等 Java 后端上线后再填，例如：

```text
NEXT_PUBLIC_API_BASE_URL=https://api.example.com
```

重要：

1. 线上前端不能用 `http://127.0.0.1:8080` 访问你的本地后端。
2. 用户浏览器里的 `127.0.0.1` 指的是用户自己的电脑。
3. 后端没部署前，线上前端只能先看页面，业务接口不可完整使用。

### 第五步：点击 Deploy

Netlify 会自动：

1. 拉取 GitHub 仓库。
2. 安装前端依赖。
3. 执行 `npm run build`。
4. 生成公网地址。

## 7. 部署成功后检查

部署完成后打开 Netlify 给你的地址，检查：

```text
[ ] 首页能打开
[ ] 登录页能打开
[ ] `/plans` 能打开
[ ] `/spaces` 能打开
[ ] 手机浏览器能打开
[ ] 页面样式正常
[ ] 没有明显白屏
```

如果登录失败，这是正常的，因为 Java 后端还没有公网地址。

## 8. 后续完整上线需要改什么

等后端部署后，需要回到 Netlify 配置：

```text
NEXT_PUBLIC_API_BASE_URL=你的 Java 后端公网地址
```

同时 Java 后端也要配置：

```text
FRONTEND_ALLOWED_ORIGINS=你的 Netlify 前端地址
```

否则 Cookie 登录会被跨域限制挡住。

## 9. 前端部署成功标准

```text
[ ] Netlify 构建成功
[ ] 得到公网前端地址
[ ] 页面能打开
[ ] 手机端能打开
[ ] 知道接口不可用是因为后端尚未部署
[ ] 已记录 Netlify 地址，准备下一步部署后端
```
