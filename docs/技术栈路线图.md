# 技术栈路线图

## 1. 阶段顺序

用户已确认：先完成第一版，再优先做 AI 增强，最后补企业工程化能力。

因此本项目阶段顺序调整为：

```text
第一阶段：MVP 基础产品
第二阶段：AI 增强
第三阶段：企业工程化与部署能力增强
```

## 2. 第一阶段：MVP 基础产品

目标：先把产品主流程跑通。

技术栈：

1. 前端：Next.js。
2. 业务后端：Spring Boot。
3. ORM：MyBatis-Plus。
4. 数据库：MySQL。
5. 缓存：Redis。
6. 图片存储：阿里云 OSS。
7. AI 服务：Python FastAPI。
8. AI 编排：LangChain。
9. 大模型：阿里云百炼 / 通义千问，模型暂定 `qwen3.5-omni-plus`。

第一阶段核心功能：

1. 地图首页。
2. 去过地区标记。
3. 旅行记录图文流。
4. 照片上传。
5. 日历计划。
6. 邀请另一半加入。
7. AI 日记润色。

## 3. 第二阶段：AI 增强

目标：让项目更贴近 AI 智能应用开发岗。

可选能力：

1. AI 旅行总结。
2. AI 行程建议。
3. 图片描述和标签生成。
4. RAG。
5. 向量数据库。
6. Embedding。
7. 语义搜索旅行记忆。
8. 多轮对话记忆。
9. 流式 AI 输出。
10. Prompt 模板管理。
11. AI 调用日志。
12. 模型可配置与模型切换。

示例能力：

```text
用户搜索“那次下雨去海边的旅行”，系统可以通过语义搜索找到相关照片和日记。
```

## 4. 第三阶段：企业工程化与部署能力增强

目标：补齐企业常见工程能力，让项目更适合求职展示。

可选能力：

1. Docker。
2. Docker Compose。
3. Nginx 反向代理。
4. Spring Security + JWT。
5. Swagger / OpenAPI。
6. 统一异常处理。
7. 统一日志。
8. 接口限流。
9. Flyway 或 Liquibase 数据库版本管理。
10. Spring Boot Actuator。
11. Prometheus + Grafana。
12. CI/CD。
13. Kubernetes。

第三阶段不要求一次性全部完成，可以按就业需要逐步补。

## 5. 执行原则

1. 第一阶段先保证产品可用。
2. 第二阶段优先做用户能感知到的 AI 能力。
3. 第三阶段再系统补企业工程能力。
4. 每阶段开始前，都需要和用户确认具体要做的功能和技术，不由 Codex 自行决定。
