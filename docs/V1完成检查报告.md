# V1 完成检查报告

检查日期：2026-06-24

## 1. 结论

当前项目已经达到 V1 本地开发闭环标准。

V1 不再只是静态前端原型，而是已经具备完整的三端链路：

1. Next.js 前端。
2. Spring Boot Java 后端。
3. Python FastAPI AI 服务。
4. MySQL 持久化。
5. Redis 短期状态、Session、限流和锁。
6. 阿里云 OSS 图片存储。

## 2. 已完成能力

### 2.1 用户与安全

1. 账号注册、登录、退出登录。
2. 用户名和账号唯一性校验。
3. 密码长度与确认密码校验。
4. HttpOnly Cookie 登录态。
5. Redis Session 白名单。
6. 登录失败次数限制：10 分钟内输错 5 次，锁定 10 分钟。
7. 接口由后端从 Cookie 解析当前用户，不信任前端传 userId。

### 2.2 空间与权限

1. 每个用户创建后拥有个人空间。
2. 支持创建情侣空间。
3. 情侣空间第一版固定 2 人。
4. 未邀请伴侣前，情侣空间不可编辑。
5. 支持空间切换，地图、日记、计划都按当前空间隔离。
6. 邀请码 1 分钟过期。
7. 邀请码使用 Redis 做短期有效期与防重复占用。

### 2.3 旅行地图与记录

1. 首页中国地图。
2. 省份下钻进入省内城市地图。
3. 上海、重庆等直辖市按城市处理。
4. 去过的城市有统一绿色标记。
5. 城市记录全部删除后，地图标记恢复普通状态。
6. 城市内支持发布图片和日记。
7. 发布、编辑、删除记录走后端接口。
8. 删除功能带确认弹窗。
9. 城市图片支持 3D 图片球和图片预览。

### 2.4 图片与 OSS

1. 图片上传接入阿里云 OSS。
2. 前端通过后端签名信息上传。
3. 图片元数据保存到 MySQL。
4. 删除图片时同步删除 OSS 文件，避免存储堆积。

### 2.5 旅行计划

1. 计划页改为 Day 列表。
2. 用户可以新增、编辑、删除每天计划。
3. 计划支持单人空间和情侣空间。
4. 当前空间切换后，计划内容按空间隔离。

### 2.6 AI 旅行规划 Agent v1

1. AI 规划按单日 Day 触发。
2. 前端通过 Java 后端调用，不直接暴露 Python AI 服务。
3. Java 后端负责登录、空间、Day 归属、额度校验。
4. Python FastAPI 负责 AI 规划生成。
5. 使用通义千问模型，模型名可配置。
6. 使用 SSE 流式输出。
7. 天气工具：15 天内使用真实天气，超过 15 天提示天气可能变化。
8. AI 结果包含上午、下午、晚上计划，以及附近推荐和提醒。
9. AI 草稿可应用到 Day。
10. 普通用户 30 天 3 次，VIP 用户 30 天 10 次。
11. Redis 防重复点击：同一用户同一天只能有一个 AI 生成任务运行。
12. Redis 短窗口限流：保护 Java、Python 和大模型接口。

### 2.7 设置与体验

1. “我”页面显示当前用户和当前空间。
2. 设置页支持修改用户名。
3. 设置页支持修改密码。
4. 设置页支持深浅色切换。
5. 设置页支持退出登录。
6. 移动端做了主要适配。

## 5. 推荐下一阶段

下一阶段建议进入 V1.1，而不是继续把 V1 无限加需求。

优先级建议：

1. 部署上线准备：Nginx、域名、HTTPS、环境变量、服务器目录结构。
2. 企业工程化增强：统一日志、全局异常码、接口文档、测试覆盖。
3. AI 能力增强：旅行记忆 RAG、Milvus、LangGraph、多工具 Agent。
4. 商业化能力：VIP 字段完善、额度后台、支付预留。
5. 跨端能力：PWA、小程序登录方案、微信登录。
