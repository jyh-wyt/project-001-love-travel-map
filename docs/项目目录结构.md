# 项目目录结构

## 1. 总体结构

本项目采用多服务结构：

```text
project-001-情侣旅行地图/
  apps/
    front/
    server/
    ai-service/
  docs/
  data/
  assets/
  output/
  logs/
  references/
```

## 2. `apps/front`

用途：

```text
Next.js 前端项目
```

建议使用工具：

```text
VS Code
```

负责内容：

1. 中国地图首页。
2. 地区旅行记录页。
3. 图文日记发布页。
4. 日历计划页。
5. 邀请码加入页面。
6. AI 日记润色按钮和交互。

前端只直接调用 Java Spring Boot 后端接口。

## 3. `apps/server`

用途：

```text
Spring Boot Java 业务后端
```

建议使用工具：

```text
IntelliJ IDEA
```

负责内容：

1. 匿名用户。
2. 情侣空间。
3. 邀请码。
4. 旅行记录。
5. 图片元数据。
6. 日历计划。
7. OSS 上传签名。
8. AI 调用权限和次数限制。
9. 调用 Python FastAPI AI 服务。
10. MySQL、Redis、OSS 的业务集成。

## 4. `apps/ai-service`

用途：

```text
Python FastAPI AI 服务
```

建议使用工具：

```text
PyCharm
```

负责内容：

1. 提供内部 AI 接口。
2. 使用 LangChain 调用阿里云百炼 / 通义千问。
3. 第一版实现 AI 日记润色。
4. 后续扩展旅行总结、行程建议、图片描述、语义搜索等能力。

该服务不直接暴露给前端，只由 Java 后端调用。

## 5. 其他目录

| 目录 | 用途 |
| --- | --- |
| `docs/` | 项目规划、架构、接口、数据库、开发文档 |
| `data/` | SQL、初始化数据、测试数据 |
| `assets/` | 图片、地图素材、设计素材 |
| `output/` | 打包产物、导出文件、最终交付物 |
| `logs/` | 项目过程记录、运行日志、问题记录 |
| `references/` | 参考资料、外部说明、引用文档 |

## 6. 本地开发打开方式

```text
VS Code 打开：apps/front
IntelliJ IDEA 打开：apps/server
PyCharm 打开：apps/ai-service
```

## 7. 本地启动顺序

后续开发时建议顺序：

```text
1. 启动 MySQL
2. 启动 Redis
3. 启动 Python FastAPI AI 服务
4. 启动 Java Spring Boot 后端
5. 启动 Next.js 前端
```
